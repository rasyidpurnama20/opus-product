# Glossary

Istilah dan singkatan yang dipakai di framework.

| Istilah | Definisi |
|---|---|
| **Artifact** | Output terdokumentasi dari sebuah stage (PRD, ADR, kode, test plan, dst). |
| **Stage** | Tahapan dalam siklus pengembangan. Framework ini punya 8 stage. |
| **DoR** | *Definition of Ready*. Syarat agar stage boleh dimulai. |
| **DoD** | *Definition of Done*. Kriteria stage boleh dianggap selesai. |
| **PRD** | *Product Requirements Document*. Apa yang dibangun & untuk siapa. |
| **ADR** | *Architecture Decision Record*. Catatan keputusan arsitektur + alasan + alternatif. |
| **OAS** | *OpenAPI Specification*. Kontrak HTTP API. |
| **DM** | *Data Model*. Definisi entitas, atribut, relasi. |
| **UIS** | *UI Spec*. Spesifikasi antarmuka pengguna. |
| **TB** | *Task Breakdown*. Daftar task yang dapat dieksekusi. |
| **TP** | *Test Plan*. Strategi pengujian. |
| **TR** | *Test Report*. Hasil eksekusi pengujian. |
| **RN** | *Release Notes*. Catatan rilis untuk user/stakeholder. |
| **RB** | *Runbook*. Prosedur operasional saat insiden / deploy. |
| **Traceability** | Kemampuan menelusuri kode ↔ artifact ↔ kebutuhan bisnis. |
| **Single Source of Truth** | Setiap fakta hanya hidup di satu artifact. |
| **Gherkin** | Format `Given / When / Then` untuk acceptance criteria. |
